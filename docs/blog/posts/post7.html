How This Inspector Works: Client-Side Bidi Analysis

How This Inspector Works: Client-Side Bidi Analysis

Published: December 22, 2025

You don’t need to trust a server to analyze suspicious text. Everything happens in your browser — instantly and privately. That is the core principle behind this tool.

When you paste text into the inspector, JavaScript immediately scans every single character looking for the twelve known bidirectional control characters. As soon as one is found, it is highlighted with a yellow badge showing its name and code point.

No Data Leaves Your Device

Unlike online phishing checkers that upload your input to a remote server, this inspector processes everything locally. No logs, no telemetry, no risk of sensitive data exposure. Your clipboard content never leaves your machine.

The detection logic lives in a single, readable JavaScript file. You can review it, fork it, or embed it in your own projects. It works offline after the first load.

Two Views for Maximum Clarity

The raw view shows every control character highlighted so you can see exactly what was written. The rendered view shows what a normal user sees — the final visual result after the browser processing. Together, they reveal deception that single-view tools miss.

Performance is critical. Even with thousands of controls, analysis completes in milliseconds using simple iteration and a lookup table — no complex regex or external libraries required.

This tool proves that powerful security doesn’t require complexity or privacy trade-offs. Sometimes the best defense is a lightweight, transparent client-side scanner running directly in the user’s browser.

Privacy and power are not mutually exclusive. This inspector proves it every day.
