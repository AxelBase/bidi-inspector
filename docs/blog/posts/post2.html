The RLO Attack: How paypal.com Becomes paypal.com

The RLO Attack: How paypal.com Becomes paypal.com

Published: December 12, 2025

One of the most effective and terrifying phishing techniques in use today doesn’t rely on lookalike domains or typos. It uses a single invisible Unicode character: U+202E, the Right-to-Left Override.

Here’s how it works. An attacker writes the text paypal.com but then inserts the RLO character immediately after the dot. Everything that follows is now rendered from right to left. So when they type moc.layap backwards, it appears as paypal.com — perfectly aligned, perfectly convincing.

A Real-World Example

Imagine receiving an email: “Your payment to paypal.com was successful.” You can view it here: paypal.com invoice_3921.exe”. Most users glance at the domain and click without hesitation. But the actual link points to a malicious server because the visible text has been visually reversed using bidirectional override.

Why It Works So Well

The human brain trusts what it sees. When a domain name looks correct, we rarely question it. Security indicators like green padlocks or HTTPS don’t help if the domain itself is fake but disguised. Even technical users can be fooled because the attack happens at the rendering layer — before any URL parsing occurs.

This technique has been documented in attacks against cryptocurrency wallets, banking portals, and software update systems. In 2024, researchers found over 8,000 active phishing pages using RLO to spoof PayPal, Apple, Microsoft, and banking brands.

The scariest part? Many antivirus tools and browser protections still don’t detect or block this method because the URL itself is technically valid — only the visual representation is deceptive.

That’s why client-side tools like this inspector are essential. They reveal what’s really in the text before your eyes or your users are tricked.

An invisible character should never have the power to steal trust. Now you know how to take that power back.
